PyDevices / palettes

Color palettes for pydisplay.

palettes is a small, portable color toolkit for pydisplay demos and apps. One entry point — get_palette() — gives you indexed wheels and cubes, Material Design swatches, and named colors without hard-coding hex in every example.

Install with MIP on MicroPython or a wheel from TestPyPI. The same import palettes path shows up in the browser gallery and on device firmware. pdwidgets builds its ColorTheme on the Material palette under the hood.

Documentation PyScript demos GitHub

Install

import mip
mip.install("palettes", index="https://PyDevices.github.io/micropython-lib/mip/PyDevices")

get_palette names

kwargs include color_depth and swapped for hardware
name What you get Useful kwargs
"default" Windows-16 style named colors (palette.RED, …) color_depth, swapped
"wheel" HSV color wheel as an indexed sequence length, saturation, color_depth, swapped
"cube" RGB color cube size, color_depth, swapped
"material_design" Material swatches + named hues (palette.RED, RED_S900, …) color_depth, swapped

Snippets

Wheel fill

Indexed colors into display_drv.

from palettes import get_palette

palette = get_palette(name="wheel", length=256, saturation=1.0)
display_drv.fill_rect(0, 0, 10, 10, palette[0])

Material named colors

Attributes on default / material_design palettes.

from palettes import get_palette

palette = get_palette(name="material_design", color_depth=16)
color = palette.RED
shade = palette.RED_S900

Hardware byte order

Match panel endianness with swapped.

palette = get_palette(
    name="wheel",
    length=256,
    color_depth=16,
    swapped=True,  # little-endian 16-bit panels
)

Used by pdwidgets

When you construct pd.Display(...), pdwidgets loads a Material palette and wraps it in ColorTheme — roughly: 

pal = get_palette(
    "material_design",
    swapped=display.needs_swap,
    color_depth=display_drv.color_depth,
)
theme = ColorTheme(pal)  # theme.background, theme.on_surface, …
